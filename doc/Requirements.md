# 로또 번호 생성기 웹 서비스 v1 요구사항

## 1. 제품 개요

본 서비스는 과거 로또 당첨 데이터의 패턴을 참고해 번호를 생성하는 혼합형 웹 애플리케이션이다. v1은 생성기 중심의 최소 MVP로 제공한다.

## 1.1 세부 기준 문서

* 생성 로직 상세: `Generation_Logic_Spec.md`
* API 상세 계약: `API_Contract_Detail.md`
* 데이터 수집 및 검증: `Data_Pipeline_and_Validation.md`
* 화면 상태 상세: `UI_State_Spec.md`
* 배포 및 운영 체크: `Deployment_and_Operations_Checklist.md`

## 2. 기능 요구사항

### 2.1 번호 생성기

* `FR-001` 사용자는 `random`, `frequency`, `mixed` 중 한 가지 전략을 선택할 수 있어야 한다.
* `FR-002` 사용자는 한 번에 1~5세트 번호를 생성할 수 있어야 한다.
* `FR-003` 각 세트는 1~45 범위의 중복 없는 6개 번호를 반환해야 한다.
* `FR-004` 선택적으로 보너스 번호를 함께 생성할 수 있어야 한다.
* `FR-005` 각 생성 결과는 간단한 근거 설명을 포함해야 한다.

### 2.2 회차 조회

* `FR-006` 사용자는 최근 회차 목록을 볼 수 있어야 한다.
* `FR-007` 사용자는 특정 회차 번호를 API로 조회할 수 있어야 한다.
* `FR-008` 각 회차는 추첨일, 메인 번호, 보너스 번호를 표시해야 한다.

### 2.3 기본 통계

* `FR-009` 사용자는 번호별 출현 빈도를 볼 수 있어야 한다.
* `FR-010` 통계는 전체 기간과 최근 10회 기준을 지원해야 한다.
* `FR-011` 통계는 메인 번호 기준으로 먼저 제공한다.

### 2.4 사용자 인터페이스

* `FR-012` 홈은 생성기 진입을 우선 노출해야 한다.
* `FR-013` 모바일과 데스크톱에서 모두 사용 가능해야 한다.
* `FR-014` 로딩 상태와 실패 상태를 표시해야 한다.

## 3. 비기능 요구사항

* `NFR-001` 앱은 데이터베이스 없이도 로컬에서 기동 가능해야 한다.
* `NFR-002` 번호 생성 응답은 일반적인 로컬 환경에서 즉시 체감 가능한 수준이어야 한다.
* `NFR-003` 입력값 검증과 기본 에러 응답 형식을 제공해야 한다.
* `NFR-004` App Router와 Route Handlers를 사용해야 한다.

## 4. 기술 요구사항

* `TR-001` Next.js 14 App Router
* `TR-002` TypeScript
* `TR-003` Tailwind CSS
* `TR-004` Prisma 스키마 제공
* `TR-005` PostgreSQL 전환 가능 구조

## 5. 데이터 요구사항

* `DR-001` 시드 데이터는 최근 회차 샘플을 포함해야 한다.
* `DR-002` `draws`와 `generated_sets` 중심 구조를 유지해야 한다.
* `DR-003` 시드 저장소와 DB 저장소를 교체할 수 있도록 추상화해야 한다.
