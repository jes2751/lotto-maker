# QA Docs

## 목적

- 디자인 QA, 기능 QA, 후속 수정 내역을 따로 모아 두는 문서 폴더입니다.
- `history.md`는 전체 변경 이력용이고, 이 폴더는 검토 결과와 근거를 다시 보기 쉽게 정리하는 용도입니다.

## 문서 목록

- `2026-04-02-design-qa.md`
  - 컨트롤룸 리디자인 이후 진행한 디자인 중심 QA 결과
  - 점검 경로, 실제 발견 이슈, 수정 내용, 최종 검증 상태 포함

## 사용 원칙

- QA를 한 번 끝내면 `history.md`에는 짧게 남기고, 자세한 내용은 이 폴더에 별도 문서로 정리합니다.
- 실제 브라우저 기준으로 확인한 내용만 QA 문서에 적습니다.
- 새 QA 문서를 추가할 때 파일명은 `YYYY-MM-DD-주제.md` 형식을 사용합니다.
