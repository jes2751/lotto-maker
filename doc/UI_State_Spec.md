# 화면 상태 명세 v1

## 1. 공통 원칙

* 모바일 우선 단일 컬럼 레이아웃을 기본으로 한다.
* 데스크톱에서는 카드와 패널을 다단으로 확장한다.
* 모든 화면은 정상 상태, 로딩 상태, 빈 상태, 오류 상태를 정의한다.
* 버튼 비활성화와 오류 메시지는 인라인으로 보여준다.
* 최소 접근성 기준:
  * 버튼과 링크는 명확한 텍스트를 가진다.
  * 색상만으로 상태를 구분하지 않는다.
  * 키보드 포커스가 보인다.

## 2. 홈 `/`

정상 상태:

* 서비스 소개
* 생성기 진입 CTA
* 최신 회차 요약
* 상위 빈도 번호 요약

빈 상태:

* 최신 회차가 없으면 `회차 데이터 준비 중` 메시지를 노출한다.
* 생성기 CTA는 계속 노출한다.

오류 상태:

* 최신 회차 또는 통계 데이터를 읽지 못하면 요약 카드 대신 `데이터를 불러오지 못했습니다.` 문구를 보여준다.

## 3. 생성기 `/generate`

정상 상태:

* 전략 선택
* 세트 수 선택
* 보너스 포함 여부 선택
* 생성 결과 목록

로딩 상태:

* 생성 요청 중 버튼을 비활성화한다.
* 버튼 텍스트는 `생성 중...`으로 바꾼다.

오류 상태:

* API 실패 시 결과 패널 상단에 인라인 오류 메시지를 표시한다.
* 마지막 성공 결과가 있으면 화면에 유지한다.

빈 상태:

* 첫 진입 시 결과가 없으면 자동 생성 또는 안내 문구 중 하나를 사용한다.
* 현재 구현 기준은 첫 진입 시 자동 생성으로 간주한다.

## 4. 회차 조회 `/draws`

정상 상태:

* 최신 회차 카드 목록
* 각 카드에 회차, 추첨일, 메인 번호, 보너스 번호 표시

빈 상태:

* 회차가 0건이면 `표시할 회차 데이터가 없습니다.` 카드 노출

오류 상태:

* 조회 실패 시 `회차 데이터를 불러오지 못했습니다.` 메시지 노출

## 5. 통계 `/stats`

정상 상태:

* 전체 기준 상위 번호
* 최근 10회 기준 상위 번호

빈 상태:

* 데이터가 0건이면 차트나 카드 대신 `통계 데이터 준비 중` 요약 상태를 노출한다.

오류 상태:

* 통계 계산 또는 조회 실패 시 `통계 데이터를 불러오지 못했습니다.` 메시지를 표시한다.

## 6. 반응형 규칙

* 모바일: 단일 컬럼, CTA 우선, 텍스트 우선
* 태블릿: 2열 카드 허용
* 데스크톱: 홈과 통계 화면에서 다단 패널 허용
