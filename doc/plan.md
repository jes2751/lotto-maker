# Lotto Maker Lab 프로젝트 플랜 v1.1

## 1. 문서 목적

이 문서는 Lotto Maker Lab의 최상위 방향 문서다.  
무엇을 만들지, 무엇을 먼저 만들지, 무엇은 나중으로 미룰지를 정리한다.

세부 기준은 아래 문서를 따른다.

- `Product_Spec.md`: 제품 요구사항, 화면 구조, 사용자 흐름
- `Design_Guide.md`: 디자인 원칙, 정보 위계, 가독성 기준
- `Technical_Spec.md`: 추천 로직, 통계, API, 데이터, 배포 기준
- `WORKFLOW.md`: 작업 순서, 역할, 로컬 작업/푸시 규칙
- `check_list.md`: 현재 달성률과 퍼블리시 전 체크 항목
- `history.md`: 실제 작업 로그

## 2. 서비스 정의

### 2-1. 서비스명

- 공식 서비스명: `Lotto Maker Lab`
- 기본 도메인: `https://lotto-maker.cloud`

### 2-2. 서비스 방향

- `과거 당첨 데이터 기반 추천 + 읽기 쉬운 통계`가 핵심이다.
- 서비스는 재미와 참고를 위한 로또 웹 서비스다.
- `당첨 보장`, `무조건 당첨` 같은 표현은 금지한다.
- 추천 결과는 어디까지나 참고용으로 제공한다.

### 2-3. 핵심 가치

- 최신 회차와 전체 회차를 빠르게 확인할 수 있다.
- 과거 당첨 데이터 기반으로 추천 번호를 생성할 수 있다.
- 번호, 회차, 통계를 한 서비스 안에서 자연스럽게 탐색할 수 있다.
- 검색 유입에 대응하는 설명형 랜딩과 가이드를 함께 제공한다.

## 3. v1 목표

- Next.js 14 App Router 기반으로 실제 배포 가능한 웹 서비스를 만든다.
- 로컬과 배포 환경에서 안정적으로 동작하는 추천/조회/통계 흐름을 확보한다.
- `Lotto Maker Lab` 브랜드와 `lotto-maker.cloud` 도메인 기준으로 메타데이터와 페이지 구조를 통일한다.
- 검색 유입과 재방문을 만들기 위한 SEO 랜딩 페이지와 가이드 허브를 함께 운영한다.

## 4. v1 핵심 범위

### 4-1. 페이지

1. 홈 `/`
2. 추천기 `/generate`
3. 회차 조회 `/draws`
4. 회차 상세 `/draws/[round]`
5. 통계 대시보드 `/stats`

7. SEO 랜딩 페이지

   - `/lotto-number-generator`
   - `/lotto-statistics`
   - `/hot-numbers`
   - `/cold-numbers`
   - `/odd-even-pattern`
   - `/sum-pattern`
   - `/recent-10-draw-analysis`

3. `GET /api/v1/draws/[round]`
4. `POST /api/v1/generate`

### 4-3. 추천 전략

- `mixed`
- `frequency`
- `random`
- `filter`

## 5. 퍼블리시 전 우선순위

### 5-1. 1차 우선순위

1. 필터 추첨기 완성도 높이기
   - 고정수
   - 제외수
   - 홀짝 조건
   - 합계 범위
   - 연속번호 허용 여부
   - 조합 수 선택
2. 종합 통계 대시보드 강화
   - 전체/최근 흐름 요약
   - 자주 나온 번호 / 적게 나온 번호
   - 홀짝 패턴
   - 합계 패턴
   - 번호 상세 드릴다운
3. 정책/신뢰 페이지
   - 개인정보처리방침
   - 이용약관
   - FAQ
   - 문의/운영 안내
4. 퍼블리시 품질 마감
   - 모바일 QA
   - 카피 정리
   - 광고 위치 점검
   - 배포 환경 데이터 확인

### 5-2. 2차 우선순위

1. 그룹 추첨기
2. 그룹 필터 추첨기
3. 번호 궁합 통계
4. 고급 통계
   - AC
   - 표준편차
   - 색상 통계
   - 패턴 통계
   - 궁합수
5. 오늘의 운세 기반 추천 번호
   - 메인 추천기와 별도 기능
   - 재미용/참고용으로 명확히 분리

## 7. 벤치마킹 반영 원칙

### 7-1. 추천기 서비스에서 배운 점

- 첫 화면에서 바로 추천기를 열 수 있어야 한다.
- 추천기 종류는 카드와 짧은 설명으로 구분하는 것이 좋다.
- 홈에서 최신 회차, 추천, 통계가 동시에 보이되 너무 복잡하지 않아야 한다.

### 7-2. 통계 서비스에서 배운 점

- 번호별 상세 통계로 자연스럽게 이동할 수 있어야 한다.
- 통계는 한 화면에 다 쏟기보다 랜딩과 상세로 나눠야 한다.
- `번호 궁합`은 감성 기능이 아니라 동반 출현 통계로 정의한다.

### 7-3. 정책/신뢰 페이지에서 배운 점

- 광고와 분석 도구를 쓰는 만큼 정책 페이지가 필요하다.
- footer에서 정책 페이지로 바로 이동할 수 있어야 한다.
- 정책 문구에는 실제 사용하는 도구만 적는다.

## 8. 브랜드 및 메타데이터 기준

- 서비스명은 `Lotto Maker Lab`으로 통일한다.
- 로고형 표기는 `LOTTO MAKER LAB`을 사용한다.
- 페이지 제목 형식은 `페이지명 | Lotto Maker Lab`으로 맞춘다.
- 도메인은 항상 `https://lotto-maker.cloud` 기준으로 다룬다.

## 9. 성공 조건

### 9-1. 제품

- 추천기, 회차 조회, 통계, 랜딩 페이지가 모두 정상 동작한다.
- 사용자가 홈에서 1~2번 클릭 안에 원하는 기능으로 이동할 수 있다.

### 9-2. 운영

- 광고는 설정 전까지 노출되지 않는다.
- Google Analytics가 동작한다.
- Search Console 제출이 가능한 구조를 갖춘다.

### 9-3. 품질

- `npm test` 통과
- `npm run build` 통과
- 모바일과 데스크톱에서 핵심 페이지가 깨지지 않는다.

## 10. 현재 판단

- 초기 MVP는 거의 완료 상태다.
- 퍼블리시 전 목표 기준으로는 `필터 추첨기 고도화`, `종합 통계`, `정책/신뢰 페이지`가 핵심 남은 과제다.
- 현재 단계에서는 기능 추가보다 `퍼블리시 가능한 완성도`를 올리는 것이 중요하다.
