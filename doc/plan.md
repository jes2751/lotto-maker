# 로또 서비스 v1 플랜

## 1. 문서 체계

이 문서는 프로젝트의 최상위 기준 문서다. 상세 기준은 아래 문서를 따른다.

* `Product_Spec.md`: 제품 기획, 화면, 사용자 흐름
* `Design_Guide.md`: 웹디자인 기준
* `Technical_Spec.md`: 생성 로직, API, 데이터, 테스트 기준
* `WORKFLOW.md`: 역할, 작업 순서, 기록, 커밋/푸시 규칙
* `history.md`: 실제 작업 로그

기존 세부 문서는 `archive/`에 보관한다.

## 2. 서비스 방향

* 서비스 포지셔닝은 `혼합형 로또 웹 서비스`다.
* 첫 사용자 경험의 중심은 `번호 추천기`에 둔다.
* 추천의 핵심 기준은 기존 당첨 데이터다.
* 통계와 회차 조회는 추천 결과가 어떤 과거 데이터 흐름을 참고했는지 이해시키는 보조 기능으로 제공한다.
* `당첨 보장`, `무조건 적중` 같은 표현은 금지한다.
* 무료 서비스 기준으로 운영하고, 로그인 없이 핵심 기능을 사용할 수 있게 한다.

## 3. v1 목표

* Next.js 14 App Router 기반으로 즉시 실행 가능한 웹 서비스를 제공한다.
* 과거 당첨 데이터 기반 추천, 전체 회차 조회, 기본 통계를 제공한다.
* 최신 당첨 데이터를 매주 일요일 `Asia/Seoul` 기준으로 반영할 수 있는 운영 기준을 갖춘다.
* 이후 Prisma/PostgreSQL 전환이 가능한 구조를 유지한다.
* 광고는 붙일 수 있게 준비하되, 초기 성장 우선순위는 광고보다 검색 유입과 재방문 확보에 둔다.

## 6. 퍼블리시 전 구현 우선순위

### 6-1. 1차 우선순위

* `필터 추첨기`: 제외수, 고정수, 홀짝 조건, 합계 범위, 연속번호 허용 여부를 포함하는 실사용 중심 확장
* `종합 통계 대시보드`: 전체/최근 비교, 홀짝 통계, 연속 미출현 통계, 연속번호 통계를 한 화면에서 보는 구조
* `통계 상세 확장`: 번호 상세에서 출현 추이, 최근 포함 회차, 기초 회귀 통계를 더 쉽게 탐색하도록 보강
* `퍼블리시 품질 마감`: 모바일 QA, 카피 정리, 광고 위치 점검, 배포 환경 데이터 확인

### 6-2. 2차 우선순위

* `그룹 추첨기`
* `그룹 필터 추첨기`
* `고급 통계 묶음`: 앞수합, 뒷수합, AC, 표준편차, 패턴, 색상, 궁합수 통계
* `재미 기능`: 룰렛형 추첨, 수동 추첨, 시뮬레이션

### 6-3. 퍼블리시 기준선

* 추천기: 기본 추천기 + 필터 추첨기
* 통계: 종합 통계 + 번호 상세 통계 + 주요 패턴 통계 일부
* 회차: 전체 회차 조회 + 번호 포함 회차 검색
* 운영: 주간 데이터 반영 기준 + 광고 비노출/노출 분기 + 배포 후 점검 기준

### 6-4. 실행 순서

1. 필터 추첨기 구현
2. 종합 통계 대시보드 구현
3. 연속 미출현/홀짝/연속번호 통계 보강
4. 퍼블리시 QA와 광고 배치 확인
5. 이후 그룹 추첨기와 고급 통계 확장

## 7. SEO 및 재방문 로드맵

### 7-1. 기본 원칙

* 초기 성장 우선순위는 `광고 수익화`보다 `검색 유입 확보`와 `재방문 이유 만들기`에 둔다.
* 키워드 하나당 페이지를 무분별하게 늘리기보다, 사용자 의도 하나당 가치 있는 랜딩 페이지 하나를 만든다.
* 검색용 페이지는 기능 복제형이 아니라 회차 데이터, 통계, 설명을 함께 제공하는 페이지로 설계한다.
* 추천기와 통계는 도구 역할을 하고, 회차 분석과 설명형 콘텐츠는 검색 유입 역할을 맡는다.

### 7-2. 기술 SEO 우선 작업

1. Google Search Console 연결
2. `sitemap.xml`과 `robots.txt` 제공
3. 페이지별 `title`과 `meta description` 개별 설정
4. 회차 페이지, 통계 페이지, 콘텐츠 페이지에 맞는 구조화 데이터 검토
5. URL 구조를 사람이 읽을 수 있고 주제가 드러나는 형태로 유지

### 7-3. 검색 유입용 랜딩 페이지 후보

* `/lotto-number-generator`
* `/lotto-statistics`
* `/latest-lotto-results`
* `/draw/[round]`
* `/hot-numbers`
* `/cold-numbers`
* `/odd-even-pattern`
* `/sum-pattern`
* `/recent-10-draw-analysis`

### 7-4. 우선 타깃 검색 의도

* 로또 번호 추천
* 로또 번호 생성기
* 로또 통계
* 로또 자주 나온 번호
* 로또 최근 당첨번호
* 로또 회차 조회
* 로또 번호 조합 추천
* 로또 번호 분석

### 7-5. 메타데이터 방향

* 홈: `로또 번호 생성기 | 과거 당첨 데이터 기반 추천 - LOTTO LAB`
* 통계: `로또 통계 | 자주 나온 번호·최근 흐름 분석 - LOTTO LAB`
* 최신 결과: `최신 로또 당첨번호 | 회차별 결과 확인 - LOTTO LAB`
* 회차 상세: `{round}회 로또 당첨번호 | 번호 조합과 통계 확인 - LOTTO LAB`
* 설명형 콘텐츠는 제목과 설명에 회차 또는 분석 주제를 직접 포함한다.

### 7-6. 콘텐츠 확장 방향

* `로또 번호 생성기, 완전 랜덤과 통계형의 차이`
* `최근 20회 기준 자주 나온 번호 정리`
* `로또 번호 고를 때 사람들이 많이 보는 통계 5가지`
* `{round}회 당첨번호 간단 분석`
* `로또 번호 조합에서 홀짝 비율은 어떻게 볼까?`
* `로또 번호 추천은 왜 재미용으로 봐야 할까?`

### 7-7. 주간 운영 루틴

* 금요일 밤: 이번 주 참고용 추천 번호와 최근 흐름 정리
* 토요일 낮: 추첨 전 빠른 추천과 최근 통계 강조
* 토요일 추첨 직후: 최신 회차 결과 업데이트
* 일요일: 이번 회차 간단 분석과 다음 회차 참고 통계 발행

### 7-8. 성장 구현 우선순위

1. Search Console, sitemap, 메타데이터 정리
2. `latest`, `hot`, `cold`, `recent-10` 랜딩 페이지 추가
3. 회차별 분석 템플릿 추가
4. 설명형 콘텐츠 5~10개 작성 루틴 구축
5. 검색 유입 데이터 확인 후 확장 페이지 우선순위 재조정

## 8. 장기 확장 방향

### 8-1. 추천기 확장

* 필터 추첨기
* 그룹 추첨기
* 그룹 필터 추첨기
* 통계 기반 다양한 조합 옵션
* 수동 추첨 보조 기능

### 8-2. 통계 확장

* 종합 통계
* 번호별 출현 통계
* 연속 미출현 통계
* 홀짝 통계
* 연속번호 통계
* 이월수/회귀 통계
* 앞수합/뒷수합/첫수합/끝수합 통계
* AC, 표준편차, 궁합수, 패턴, 색상 통계

### 8-3. 확장 원칙

* `데이터 기반 추천`이라는 핵심 방향을 해치지 않는 범위에서만 기능을 추가한다.
* 경쟁 서비스 기능을 그대로 복제하지 않고, 웹 사용성에 맞게 다시 설계한다.
* 검색 유입용 페이지와 도구형 페이지가 서로 트래픽을 주고받는 구조를 만든다.

## 10. 성공 조건

* 핵심 문서가 서로 충돌 없이 유지된다.
* `Product_Spec.md`만으로 사용자 흐름과 화면 상태를 이해할 수 있다.
* `Design_Guide.md`만으로 UI 기준을 재현할 수 있다.
* `Technical_Spec.md`만으로 개발자가 구현을 시작할 수 있다.
* `WORKFLOW.md` 기준으로 작업, 검증, 기록, 푸시가 관리된다.
* 퍼블리시 이후 검색 유입용 랜딩 페이지와 설명형 콘텐츠 확장 방향이 문서에 정리되어 있다.
