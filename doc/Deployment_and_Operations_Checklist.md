# 배포 및 운영 체크리스트 v1

## 1. 사전 준비

* `main` 브랜치 기준으로 작업한다.
* 워크트리가 깨끗한지 확인한다.
* `doc/history.md`에 이번 변경과 검증 결과를 기록한다.
* `.env.example`이 현재 코드 요구사항과 일치하는지 확인한다.

## 2. 로컬 검증

최소 검증 항목:

* `npm install`
* `npm test`
* `npm run dev`

추가 확인:

* 홈, 생성기, 회차 조회, 통계 페이지 렌더링
* 생성 API 응답 확인
* 회차 API와 통계 API 응답 확인

## 3. 커밋 및 푸시

* 관련 문서와 코드가 함께 반영되었는지 확인한다.
* 커밋 메시지는 `feat:`, `fix:`, `docs:`, `chore:`, `refactor:`, `test:` 중 하나를 사용한다.
* 검증이 통과한 뒤에만 `main`으로 푸시한다.

## 4. 배포 후 확인

* 주요 페이지 응답 확인
* API 정상 응답 확인
* 시드 데이터 기준 최신 회차 표시 확인
* 기본 통계와 생성기 동작 확인

## 5. 실패 시 대응

* 검증 실패 시 푸시하지 않는다.
* 실패 원인과 현재 상태를 `doc/history.md`에 남긴다.
* 긴급 수정은 작은 단위 커밋으로 분리한다.
* 배포 후 이상이 있으면 이전 정상 커밋 기준으로 원인 분석 후 수정한다.
