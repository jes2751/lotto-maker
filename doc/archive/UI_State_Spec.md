# 화면 상태 명세 v1

## 1. 공통 원칙

* 모바일 우선 단일 컬럼 레이아웃을 기본으로 한다.
* 데스크톱에서는 카드와 패널을 다단으로 확장한다.
* 모든 화면은 정상 상태, 로딩 상태, 빈 상태, 오류 상태를 정의한다.

## 2. 홈 `/`

* 최신 회차가 없으면 `회차 데이터 준비 중` 메시지와 생성기 CTA 노출
* 데이터 조회 실패 시 오류 문구 표시

## 3. 생성기 `/generate`

* 요청 중 버튼 비활성화
* 실패 시 인라인 에러 표시
* 마지막 성공 결과 유지

## 4. 회차 조회 `/draws`

* 결과가 없으면 `표시할 회차 데이터가 없습니다.` 카드 노출
* 조회 실패 시 오류 메시지 노출

## 5. 통계 `/stats`

* 데이터가 0건이면 `통계 데이터 준비 중` 요약 상태 노출
* 통계 조회 실패 시 오류 메시지 표시

## 6. 반응형 규칙

* 모바일: 단일 컬럼
* 태블릿: 2열 카드 허용
* 데스크톱: 다단 패널 허용
