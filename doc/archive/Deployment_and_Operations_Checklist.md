# 배포 및 운영 체크리스트 v1

## 1. 사전 준비

* `main` 브랜치 기준으로 작업한다.
* 워크트리가 깨끗한지 확인한다.
* `doc/project/history.md`에 이번 변경과 검증 결과를 기록한다.
* `.env.example`이 현재 코드 요구사항과 일치하는지 확인한다.

## 2. 로컬 검증

* `npm install`
* `npm test`
* `npm run dev`

## 3. 커밋 및 푸시

* 관련 문서와 코드가 함께 반영되었는지 확인한다.
* 커밋 메시지는 규칙 접두어를 사용한다.
* 검증이 통과한 뒤에만 `main`으로 푸시한다.

## 4. 배포 후 확인

* 주요 페이지 응답 확인
* API 정상 응답 확인
* 시드 데이터 기준 최신 회차 표시 확인

## 5. 실패 시 대응

* 검증 실패 시 푸시하지 않는다.
* 실패 원인과 현재 상태를 `doc/project/history.md`에 남긴다.
