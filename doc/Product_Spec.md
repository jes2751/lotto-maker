# Product Spec v1.1

## 1. 문서 목적

이 문서는 Lotto Maker Lab의 제품 기준서다.  
사용자에게 무엇을 보여주고, 어떤 흐름으로 쓰게 만들지 정의한다.

## 2. 제품 정의

### 2-1. 한 줄 정의

Lotto Maker Lab은 과거 당첨 데이터 기반 추천, 회차 조회, 통계를 쉽게 탐색할 수 있는 무료 로또 웹 서비스다.

### 2-2. 핵심 포지션

- `추천기`와 `통계`를 함께 제공한다.
- `최신 회차 확인 -> 통계 탐색 -> 추천기 사용` 흐름이 자연스럽게 이어져야 한다.
- 결과는 참고용이며, 확률을 보장하지 않는다.

## 3. 타깃 사용자

1. 최신 회차를 빠르게 확인하려는 사용자
2. 번호 흐름과 통계를 참고하려는 사용자
3. 직접 번호를 고르기보다 추천기를 이용하고 싶은 사용자
4. 검색을 통해 `로또 번호 추천`, `로또 통계`, `최신 로또 결과`를 찾는 사용자

## 4. 핵심 사용자 흐름

### 4-1. 빠른 추천 흐름

홈 -> 추천기 -> 전략 선택 -> 결과 확인 -> 번호 상세 통계 확인

### 4-2. 최신 결과 흐름

홈/최신 결과 -> 회차 상세 -> 회차 분석 -> 통계 허브

### 4-3. 검색 유입 흐름

SEO 랜딩 페이지 -> 핵심 설명 -> 관련 통계/추천기로 이동

## 5. 핵심 기능 요구사항

### 5-1. 추천기

- 사용자는 `mixed`, `frequency`, `random`, `filter` 전략 중 선택할 수 있어야 한다.
- 추천 결과는 6개 번호를 중복 없이 제공해야 한다.
- 선택적으로 보너스 번호를 함께 보여줄 수 있어야 한다.
- 생성 이유 문구를 간단히 제공해야 한다.

### 5-2. 필터 추첨기

- 사용자는 아래 조건을 직접 입력할 수 있어야 한다.
  - 고정수
  - 제외수
  - 홀짝 조건
  - 합계 범위
  - 연속번호 허용 여부
  - 조합 수
- 화면에는 선택 개수와 조건 요약이 보여야 한다.

### 5-3. 회차 조회

- 전체 회차를 목록으로 탐색할 수 있어야 한다.
- 회차 번호로 검색할 수 있어야 한다.
- 특정 번호가 포함된 회차를 필터링할 수 있어야 한다.

### 5-4. 회차 상세

- 회차 번호
- 추첨일
- 당첨번호 6개
- 보너스 번호
- 상금 요약
- 이전/다음 회차 이동

### 5-5. 통계

- 자주 나온 번호
- 적게 나온 번호
- 홀짝 패턴
- 합계 패턴
- 최근 10회 분석
- 번호 상세 통계

## 6. 페이지 요구사항

### 6-1. 홈

- 가장 먼저 추천기와 최신 회차가 보여야 한다.
- 빠른 이동 카드가 있어야 한다.
- 핵심 분석 링크가 있어야 한다.
- 가이드 섹션이 있어야 한다.
- 화면은 깔끔하고 과밀하지 않아야 한다.

### 6-2. 추천기

- 전략 선택이 명확해야 한다.
- 필터 전략을 분리해서 이해하기 쉽게 보여야 한다.
- 결과는 재사용하기 쉬운 형태로 보여야 한다.

### 6-3. 통계 허브

- 주요 통계로 빠르게 이동할 수 있어야 한다.
- 초보자도 어디서부터 봐야 하는지 이해할 수 있어야 한다.

### 6-4. 랜딩 페이지

- 검색 의도에 맞는 제목과 설명이 있어야 한다.
- 실제 데이터/기능과 자연스럽게 연결돼야 한다.
- 단순 키워드 페이지가 아니라 가치 있는 설명형 콘텐츠여야 한다.

## 7. 정책/운영 요구사항

- 개인정보처리방침
- 이용약관
- FAQ
- 문의/운영 안내
- 광고 안내

Google Analytics와 Google AdSense를 쓰는 만큼, 관련 고지가 실제 페이지에 반영되어야 한다.

## 8. 광고/분석 원칙

- 광고는 핵심 CTA 앞을 가리지 않아야 한다.
- AdSense 설정 전까지 광고는 비노출이어야 한다.
- Analytics는 페이지 흐름을 방해하지 않게 삽입한다.

## 9. 제외 범위

아래는 퍼블리시 전 필수 기능이 아니다.

- 로그인
- 저장 기능 중심 구조
- 유료 기능
- 과도한 게임형 요소
- 개인정보를 강하게 수집하는 기능

## 10. 후속 확장

- 그룹 추첨기
- 그룹 필터 추첨기
- 번호 궁합 통계
- 고급 통계
- 오늘의 운세 기반 추천 번호

후속 기능은 메인 서비스 정체성을 흐리지 않도록 `보조 기능`으로 확장한다.
