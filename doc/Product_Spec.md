# 제품 기획서 v1

## 1. 제품 개요

로또 번호 생성기 v1은 과거 당첨 데이터의 패턴을 참고해 번호를 제안하는 혼합형 웹 서비스다. 핵심 경험은 번호 생성이며, 통계와 회차 조회는 생성 결과를 보조 설명하는 역할을 가진다.

## 2. 타깃 사용자

* 재미로 번호를 만들어보고 싶은 일반 사용자
* 최근 회차 흐름을 간단히 확인하고 싶은 사용자
* 복잡한 분석보다 가볍고 빠른 결과를 원하는 모바일 사용자

## 3. 핵심 가치 제안

* 빠르게 번호를 생성할 수 있다.
* 생성 전략과 근거가 있어 결과를 납득하기 쉽다.
* 최신 회차와 기본 통계를 한 흐름 안에서 확인할 수 있다.

## 4. 사용자 시나리오

### 사용자 시나리오 1

* 사용자는 홈에서 서비스 설명과 최신 회차 요약을 본다.
* 생성기 CTA를 눌러 번호 생성 화면으로 이동한다.
* 전략과 세트 수를 고른 뒤 결과를 확인한다.

### 사용자 시나리오 2

* 사용자는 통계 화면에서 자주 나온 번호를 확인한다.
* 회차 조회 화면에서 최근 회차 번호를 함께 비교한다.

## 5. 기능 요구사항

### 번호 생성기

* 사용자는 `random`, `frequency`, `mixed` 중 전략을 선택할 수 있어야 한다.
* 사용자는 한 번에 1~5세트 번호를 생성할 수 있어야 한다.
* 각 세트는 6개 메인 번호와 선택적 보너스 번호를 반환해야 한다.
* 각 생성 결과는 간단한 근거 문구를 포함해야 한다.

### 회차 조회

* 사용자는 최근 회차 목록을 볼 수 있어야 한다.
* 각 회차는 추첨일, 메인 번호, 보너스 번호를 포함해야 한다.

### 기본 통계

* 사용자는 번호별 출현 빈도를 볼 수 있어야 한다.
* 전체 기준과 최근 10회 기준을 지원해야 한다.
* v1 통계는 메인 번호 기준으로 제공한다.

## 6. 정보구조

```text
홈 (/)
├── 번호 생성기 (/generate)
├── 회차 조회 (/draws)
└── 기본 통계 (/stats)
```

## 7. 화면 목록

### 홈 `/`

* 서비스 소개
* 전략 요약
* 최신 회차 요약
* 생성기 이동 CTA

### 번호 생성기 `/generate`

* 전략 선택
* 세트 수 선택
* 보너스 포함 여부 선택
* 생성 결과와 근거 표시

### 회차 조회 `/draws`

* 최근 회차 카드 목록
* 각 회차 번호, 보너스, 추첨일 표시

### 기본 통계 `/stats`

* 전체 기준 상위 번호
* 최근 10회 기준 상위 번호

## 8. 사용자 플로우

1. 홈 진입
2. 생성기 이동
3. 번호 생성
4. 필요 시 회차 조회 또는 통계 화면 이동

## 9. 화면 상태 정의

### 공통

* 모바일 우선 단일 컬럼
* 데스크톱에서 다단 레이아웃 확장
* 정상, 로딩, 빈 상태, 오류 상태를 모두 정의

### 홈

* 최신 회차가 없으면 `회차 데이터 준비 중` 메시지와 생성기 CTA 노출

### 생성기

* 요청 중 버튼 비활성화
* 실패 시 인라인 에러 표시
* 마지막 성공 결과 유지

### 회차 조회

* 데이터가 없으면 `표시할 회차 데이터가 없습니다.` 카드 노출

### 통계

* 데이터가 0건이면 `통계 데이터 준비 중` 상태 노출

## 10. v1 제외 기능

* FAQ, 소개, 운영자 페이지
