# Product Spec v1.7

## 1. 서비스 개요

Lotto Maker Lab은 과거 당첨 데이터 기반 추천, 회차 조회, 통계, 생성 통계를 제공하는 무료 로또 웹 서비스다.

핵심 방향:
- 최신 결과를 빠르게 확인할 수 있어야 한다.
- 번호 생성은 바로 이해되고 바로 사용할 수 있어야 한다.
- 통계는 한 화면에서 핵심 흐름이 보이도록 구성한다.
- 생성 통계는 공개 생성 현황과 전략 성과를 보여주는 기능으로 운영한다.
- 사용자 화면에는 내부 운영 용어를 노출하지 않는다.
- 사용자 화면은 한국어 단일 표기를 기준으로 운영한다.
- 헤더 우측에는 오늘 방문 수를 보조 정보 수준으로만 작게 노출할 수 있다.

## 2. 주요 사용자 니즈

1. 최신 당첨번호를 빠르게 확인하고 싶다.
2. 지금 바로 번호를 만들어보고 싶다.
3. 어떤 번호가 자주 나왔는지 알고 싶다.
4. 이번 회차에 사람들이 어떤 방식으로 번호를 만들었는지 보고 싶다.
5. 온라인 구매와 로또 기본 정보를 한 번에 확인하고 싶다.

## 3. 핵심 페이지

### 3-1. 홈 `/`
- 목적: 최신 결과, 번호 생성, 핵심 통계로 빠르게 진입시키는 페이지
- 구성:
  - 상단 긴 가로 영역 안에서 `최신 당첨번호 -> 브랜드 소개/번호 생성기 진입`
  - 회차 조회 진입
  - 핵심 통계 요약
  - 주요 안내 링크

### 3-2. 번호 생성기 `/generate`
- 목적: 전략과 조건을 정한 뒤 번호를 직접 생성
- 규칙:
  - 첫 진입 시 자동 생성하지 않는다.
  - 기본 생성 수는 `5세트`다.
  - 보너스 번호는 항상 함께 생성한다.
  - `filter` 전략일 때만 필터 조건 패널을 노출한다.
  - 생성 결과는 공개 생성 통계에 익명으로 반영될 수 있다.

### 3-3. 회차 조회 `/draws`
- 목적: 전체 회차를 찾고 원하는 회차 상세로 이동

### 3-4. 회차 상세 `/draws/[round]`
- 목적: 해당 회차의 당첨번호와 관련 이동 링크 제공

### 3-5. 통계 `/stats`
- 목적: 자주 보는 핵심 통계를 한 화면에서 비교
- 구성:
  - 전체 회차 / 최근 10회 비교
  - 자주 나온 번호 / 적게 나온 번호
  - 평균 합계
  - 홀짝 비율
  - 최근 반복 번호

### 3-6. 생성 통계 `/generated-stats`
- 목적: 이번 회차 공개 생성 현황과 전략 성과를 보여주는 페이지
- 구성:
  - 대상 회차
  - 이번 회차 공개 생성 수
  - 최근 평가 회차
  - 전략 성과
  - 적중 분포
  - 최근 생성 번호 일부

### 3-7. 온라인 구매 안내 `/lotto-buy-guide`
- 목적: 온라인 구매 가능 여부와 로또 기본 정보를 한 번에 안내
- 구성:
  - 온라인 구매 가능 여부
  - 구매 한도
  - 판매 시간
  - 추첨 시간
  - 기본 유의사항
  - 동행복권 메인페이지 바로가기
- 원칙:
  - 동행복권 공식 안내 기준으로 작성한다.
  - 법률/정책 확정 표현이 아니라 `공식 안내 기준` 문구를 사용한다.
  - 비공식 판매처나 우회 방법을 안내하지 않는다.

### 3-8. 가이드 및 헬프 센터
- 목적: SEO 최적화 및 사용자 궁금증 해소
- 주요 페이지:
  - `/guides`, `/guides/[slug]`: 로또 생성 대 랜덤 비교, 빈도 분석 등 읽을거리 제공
  - `/faq`: 자주 묻는 질문
  - `/privacy`, `/terms`, `/policies/ads`: 개인정보 처리, 이용약관, 광고 정책
  - `/contact`: 문의
- 원칙:
  - 핵심 서비스(번호 생성, 홈)의 동선을 방해하지 않도록 푸터나 서브 메뉴에 배치한다.

## 4. 번호 생성 전략

### 4-1. mixed
- 전체 빈도와 최근 흐름을 함께 반영하는 균형형 추천

### 4-2. frequency
- 자주 나온 번호 흐름을 더 강하게 반영하는 추천

### 4-3. random
- 조건 없이 무작위 생성

### 4-4. filter
- 고정수, 제외수, 홀짝, 합계, 연속번호 허용 여부를 조합해 맞춤형 생성

## 5. 정보 페이지 운영 원칙

- 정보 페이지는 사용자에게 실제 도움이 되는 내용을 먼저 제공한다.
- `검색 유입`, `랜딩`, `재방문` 같은 내부 용어는 사용자 화면에 쓰지 않는다.
- 온라인 구매 안내 페이지는 홍보성 문구보다 공식 정보 요약과 안전한 안내를 우선한다.
- 관련 페이지와 자연스럽게 연결한다.
  - 번호 생성기
  - 최신 결과
  - FAQ

## 6. 퍼블리시 우선순위

1. 핵심 화면 완성도
2. 생성 통계 안정화
3. 당첨번호 동기화 운영
4. 온라인 구매 안내와 기본 정보 강화
5. Search Console 등록 및 색인
