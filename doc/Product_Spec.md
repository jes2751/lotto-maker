# 제품 기획서 v1

## 1. 제품 개요

로또 번호 생성기 v1은 과거 당첨 데이터의 패턴을 바탕으로 번호를 추천하는 혼합형 웹 서비스다. 핵심 경험은 데이터 기반 번호 추천이며, 통계와 회차 조회는 추천 결과를 보조 설명하는 역할을 가진다.

## 2. 타깃 사용자

* 재미로 번호를 만들어보고 싶은 일반 사용자
* 최근 회차 흐름을 간단히 확인하고 싶은 사용자
* 복잡한 분석보다 가볍고 빠른 결과를 원하는 모바일 사용자

## 3. 핵심 가치 제안

* 빠르게 번호를 생성할 수 있다.
* 기존 당첨 데이터를 바탕으로 한 추천 전략과 근거가 있어 결과를 납득하기 쉽다.
* 최신 회차와 기본 통계를 한 흐름 안에서 확인할 수 있다.

## 4. 사용자 시나리오

### 사용자 시나리오 1

* 사용자는 홈에서 서비스 설명과 최신 회차 요약을 본다.
* 생성기 CTA를 눌러 번호 생성 화면으로 이동한다.
* 과거 당첨 데이터 기반 전략과 세트 수를 고른 뒤 추천 결과를 확인한다.

### 사용자 시나리오 2

* 사용자는 통계 화면에서 자주 나온 번호를 확인한다.
* 회차 조회 화면에서 최근 회차 번호를 함께 비교한다.

## 5. 기능 요구사항

### 번호 생성기

* 사용자는 과거 당첨 데이터 기반 추천 전략인 `frequency`, `mixed`를 선택할 수 있어야 한다.
* 사용자는 비교용 옵션으로 `random`도 선택할 수 있어야 한다.
* 기본 선택 전략은 `mixed`여야 한다.
* 사용자는 한 번에 1~5세트 번호를 추천받을 수 있어야 한다.
* 각 세트는 6개 메인 번호와 선택적 보너스 번호를 반환해야 한다.
* 각 추천 결과는 어떤 과거 당첨 데이터 기준을 참고했는지 간단한 근거 문구를 포함해야 한다.

### 회차 조회

* 사용자는 최근 회차 목록을 볼 수 있어야 한다.
* 각 회차는 추첨일, 메인 번호, 보너스 번호를 포함해야 한다.

### 기본 통계

* 사용자는 번호별 출현 빈도를 볼 수 있어야 한다.
* 전체 기준과 최근 10회 기준을 지원해야 한다.
* v1 통계는 메인 번호 기준으로 제공한다.

## 6. 정보구조

```text
홈 (/)
├── 번호 생성기 (/generate)
├── 회차 조회 (/draws)
└── 기본 통계 (/stats)
```

## 7. 화면 목록

### 홈 `/`

* 서비스 소개
* 과거 당첨 데이터 기반 추천 전략 요약
* 최신 회차 요약
* 지난 회차 당첨번호 카드
* 생성기 이동 CTA

### 번호 생성기 `/generate`

* 전략 선택
* 세트 수 선택
* 보너스 포함 여부 선택
* 추천 결과와 과거 데이터 기준 근거 표시
* 광고가 있더라도 생성 흐름을 가리지 않는 위치에만 노출

### 회차 조회 `/draws`

* 최근 회차 카드 목록
* 각 회차 번호, 보너스, 추첨일 표시

### 기본 통계 `/stats`

* 전체 기준 상위 번호
* 최근 10회 기준 상위 번호

## 8. 사용자 플로우

1. 홈 진입
2. 생성기 이동
3. 과거 당첨 데이터 기반 추천 실행
4. 필요 시 회차 조회 또는 통계 화면 이동

## 9. 홈 화면 우선 배치 기준

홈 첫 화면의 기본 순서는 아래와 같다.

1. 서비스 소개와 생성기 CTA
2. 지난 회차 당첨번호 카드
3. 추천 전략 요약
4. 추가 통계 또는 보조 정보
## 10. 화면 상태 정의

### 공통

* 모바일 우선 단일 컬럼
* 데스크톱에서 다단 레이아웃 확장
* 정상, 로딩, 빈 상태, 오류 상태를 모두 정의

### 홈

* 최신 회차가 없으면 `회차 데이터 준비 중` 메시지와 생성기 CTA 노출
* 최신 회차 반영이 지연되면 `최신 회차 데이터 반영 중입니다.` 문구를 노출한다.
* 데이터 조회 실패 시 `지난 회차 데이터를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요.` 문구를 노출한다.

### 생성기

* 요청 중 버튼 비활성화
* 실패 시 인라인 에러 표시
* 마지막 성공 결과 유지
* 추천 실패 시 `추천 번호를 생성하지 못했습니다. 잠시 후 다시 시도해 주세요.` 문구를 노출한다.

### 회차 조회

* 데이터가 없으면 `표시할 회차 데이터가 없습니다.` 카드 노출
* 조회 실패 시 `회차 데이터를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요.` 문구를 노출한다.

### 통계

* 데이터가 0건이면 `통계 데이터 준비 중` 상태 노출
* 통계 조회 실패 시 `통계 데이터를 불러오지 못했습니다. 잠시 후 다시 시도해 주세요.` 문구를 노출한다.

## 11. 광고 운영 기준

* Google AdSense를 붙이더라도 핵심 기능 사용 전에 강제 상호작용을 요구하지 않는다.
* 홈, 생성기, 통계처럼 콘텐츠 흐름 사이에 자연스럽게 들어가는 인라인 광고만 우선 고려한다.
* 광고는 추천 결과, 지난 회차 카드, 통계 카드의 핵심 정보를 가리지 않아야 한다.
* 모바일에서는 첫 화면의 핵심 CTA 위에 광고를 두지 않는다.

## 12. 운영 데이터 반영 기준

* 최신 당첨 데이터는 매주 일요일 `Asia/Seoul` 기준으로 자동 반영한다.
* 자동 반영은 마지막 저장 회차보다 신규 회차가 있을 때만 수행한다.
* 반영 실패 시 기존 데이터는 유지한다.
* 사용자 화면에는 마지막 정상 반영 데이터만 노출한다.
