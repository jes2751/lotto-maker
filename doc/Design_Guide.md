# 디자인 가이드 v1

## 1. 디자인 목표

* 생성기 중심의 빠른 사용 경험
* 복잡한 분석 도구보다 가볍고 선명한 정보 전달
* 모바일에서도 답답하지 않은 카드 중심 인터페이스

## 2. 톤앤매너

* 데이터 기반이지만 무겁지 않다.
* 재미 서비스이되 조잡하지 않다.
* 밝고 선명한 포인트를 쓰되 도박성 과장 표현은 피한다.

## 3. 컬러 시스템

* 기본 배경: 짙은 남청 계열
* 기본 텍스트: 밝은 회백색
* 포인트 컬러: 주황, 민트 계열
* 상태 컬러:
  * 성공: 명도 높은 포인트 컬러
  * 오류: 붉은 계열
  * 비활성: 저채도 회색 계열

## 4. 타이포그래피

* 제목은 강한 대비와 높은 가독성을 우선한다.
* 본문은 모바일 가독성을 우선한다.
* 숫자 정보는 카드 안에서 빠르게 읽히도록 시각적 위계를 준다.

## 5. 레이아웃 규칙

* 모바일 우선 단일 컬럼을 기본으로 한다.
* 카드와 패널 간 간격은 충분히 확보한다.
* 데스크톱에서만 다단 패널 구성을 허용한다.

## 6. 공통 컴포넌트 규칙

### 버튼

* 주 CTA는 가장 눈에 띄는 포인트 컬러 사용
* 비활성 상태는 명확히 구분

### 카드

* 회차, 결과, 통계는 모두 카드형 표현 사용
* 카드 안에서 제목, 보조 설명, 핵심 숫자 순으로 위계를 둔다

### 번호 볼

* 번호 구간별 색상 차이를 둘 수 있다.
* 색상만으로 의미를 전부 전달하지 않도록 숫자를 크게 표시한다.

## 7. 홈 화면 구성 규칙

* 홈에는 지난 회차 당첨번호를 한눈에 볼 수 있는 요약 카드가 반드시 포함되어야 한다.
* 지난 회차 카드에는 최소한 `회차`, `추첨일`, `메인 번호 6개`, `보너스 번호`를 표시한다.
* 지난 회차 당첨번호 카드는 생성기 CTA와 너무 멀어지지 않게 첫 화면 주요 영역 안에 배치한다.
* 카드 안에서는 회차 정보, 날짜, 번호 볼 순으로 시각적 위계를 둔다.
* 데이터가 아직 없으면 번호 대신 `회차 데이터 준비 중` 상태를 노출한다.

## 8. 반응형 기준

* 모바일: 단일 컬럼, CTA 우선, 짧은 요약 우선
* 태블릿: 2열 카드 허용
* 데스크톱: 홈과 통계에 다단 패널 허용

## 9. 상태 표현 규칙

### 로딩 상태

* 버튼 텍스트 변화와 비활성화 사용
* 필요한 경우 인라인 로딩 상태 사용

### 빈 상태

* 데이터 준비 중 또는 결과 없음 메시지를 카드 안에서 안내
* 다음 행동으로 이어질 CTA를 함께 노출

### 오류 상태

* 인라인 오류 메시지를 우선 사용
* 사용자가 다시 시도할 수 있는 문맥을 제공

## 10. 접근성 최소 기준

* 버튼과 링크는 명확한 텍스트를 가진다.
* 키보드 포커스가 보여야 한다.
* 색상만으로 상태를 구분하지 않는다.

## 11. 금지 사항

* `당첨 보장`, `확실한 적중` 같은 표현 금지
* 과도하게 번쩍이는 시각 효과 금지
* 모바일에서 읽기 어려운 복잡한 다단 레이아웃 금지
