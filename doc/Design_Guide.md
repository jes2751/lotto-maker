# Design Guide v1.8

## 1. 방향 (Premium UI/UX)

- 첫 화면에서 가장 먼저 보여줄 정보는 `최신 당첨번호`, `번호 생성`, `핵심 통계`다.
- 사용자가 바로 행동할 수 있어야 하므로 긴 설명보다 짧은 제목, 짧은 본문, 명확한 버튼이 우선이다.
- 내부 운영 언어는 화면에 노출하지 않는다. (예: 검색 유입, 재방문, 랜딩)
- **배경 및 테마**: 정적인 색상이 아닌, 은은하게 움직이는 다이내믹 메쉬(Mesh) 그레이디언트 적용으로 살아 숨쉬는 감각적인 공간감을 제공한다.
- **컴포넌트 텍스처**: 배경을 투영하는 고급스러운 글래스모피즘(Glassmorphism)과 마이크로 애니메이션을 도입하여 전반적인 심미성을 극대화한다.

## 2. 타이포

### 서비스 전체 기본

- 기본 폰트: `Noto Sans KR`, 영문/숫자 특화 폰트로 `Outfit` 등 모던 기하학 폰트 조합 적용
- 본문 기본 두께: `500`
- 제목 기본 두께: `700~800`
- 영문 로고는 자간을 살짝 주되 과하게 벌리지 않는다.

### 글씨 크기 기준

- 메인 제목:
  - desktop: `clamp(1.95rem, 3vw, 3rem)`
  - mobile: `2rem` 전후
  - line-height: `1.14 ~ 1.2`
  - weight: `800`
- 섹션 제목:
  - desktop: `clamp(1.3rem, 2vw, 1.8rem)`
  - mobile: `1.25rem ~ 1.5rem`
  - line-height: `1.2 ~ 1.28`
  - weight: `700`
- 본문 큰 텍스트:
  - `1rem ~ 1.0625rem`
  - line-height: `1.75 ~ 1.85`
- 본문 기본:
  - `0.95rem ~ 1rem`
  - line-height: `1.7 ~ 1.8`
- 보조 텍스트:
  - `0.875rem ~ 0.9rem`
  - line-height: `1.6 ~ 1.7`
- eyebrow:
  - `0.875rem`
  - uppercase
  - tracking은 과하지 않게 `0.1em` 전후

### 타이포 원칙

- 제목은 되도록 한 줄에 보이게 하고, 안 되면 두 줄 안에서 끝낸다.
- 의미 없는 강한 확대는 피한다.
- 밝은 배경에서는 텍스트 대비를 더 진하게 잡는다.
- 작은 민트색 라벨은 너무 작게 쓰지 않는다.

## 3. 레이아웃

- 콘텐츠 최대 폭: `max-w-6xl`
- 주요 화면 상단 여백: `py-12`
- 패널 간 기본 간격: `gap-6` 또는 `gap-8`
- 패널 내부 패딩:
  - desktop: `p-6 ~ p-7`
  - mobile: `p-5 ~ p-6`
- 패널 모서리:
  - `24px` 전후
- 패널 구분:
  - 너무 많은 작은 카드보다 큰 카드 몇 개가 낫다.
- 글래스모피즘 및 인터랙션 강화:
  - 패널 표면은 정교한 내부 테두리(Inset shadow)와 강력한 블러 효과(`backdrop-blur-xl` 이상)를 적용해 투명하면서도 요소간의 깊이감을 조성한다.
  - 주요 CTA 버튼은 마우스 오버 시 부드러운 스케일 업(scale-up) 효과와 은은한 발광(Glow) 효과를 제공한다.

## 3-1. 핵심 요소 (로또 공)
- **입체감(3D) 부여**: 단순 평면 색상이 아닌, 방사형 그레이디언트(Radial Gradient)와 입체적인 내부/외부 그림자를 조합해 실제 구슬이나 칩 같은 질감을 표현한다.
- **시각적 반응**: 마우스 호버 시 공이 살짝 튀어 오르는 인터랙션을 통해 생동감을 높인다.
- **색상 채도 고급화**: 기존 단순 원색 계열을 조금 더 깊이가 있거나 세련된 네온 컬러톤(예: 딥 블루, 마젠타 핑크, 에메랄드 등)으로 재조정한다.

## 4. 홈 화면

- 첫 번째 패널:
  - `회차 당첨번호`
  - 회차, 날짜, 번호, 상세 링크만 보여준다.
- 두 번째 패널:
  - `LOTTO MAKER LAB`
  - 메인 제목 한 줄 또는 두 줄
  - 짧은 설명
  - CTA 2개 이내
- 그 아래:
  - 빠른 시작
  - 핵심 요약
  - 가이드 / 서비스 안내
- 홈에서는 섹션 수를 늘리지 않는다.

## 5. 생성기

- 상단 2단은 좌우 패널 끝선이 맞아야 한다.
- 오른쪽 끝에 애매한 빈 여백이 남으면 안 된다.
- 전략 선택은 한 화면에서 가로 흐름으로 비교되게 한다.
- `filter` 전략일 때만 필터 입력 UI를 보여준다.
- 생성 결과가 없을 때도 화면이 무겁지 않게 유지한다.
- 보너스 번호는 항상 포함한다.

## 6. 통계

- 통계 화면은 `요약 -> 핵심 번호 -> 비교 -> 차트 -> 해석` 순으로 읽히게 한다.
- 숫자 카드 제목은 한 줄에 가깝게 짧게 유지한다.
- 칩 링크는 너무 작지 않게 하고, 클릭 가능하다는 느낌을 줘야 한다.

## 7. 생성 통계

- 생성 통계는 커뮤니티처럼 보이면 안 된다.
- `전략 성과`, `현재 회차 현황`, `적중 분포`, `최근 생성 번호` 순으로 정리한다.
- 적중 분포 카드가 억지로 세로로 늘어나지 않게 한다.
- 전략 점유율, 최고 적중 수, 많이 생성된 번호는 한눈에 보여야 한다.

## 8. 헤더와 푸터

- 헤더:
  - 로고 > 설명 > 내비게이션 순서
  - 설명은 한 줄 또는 짧은 두 줄
  - 메뉴는 본문보다 약간 크게
- 푸터:
  - 소개 영역 + 링크 영역
  - 링크는 `서비스`, `정책`으로 나누되 박스는 하나로 유지
  - 푸터는 정보 정리용이지 시선을 끄는 영역이 아니다

## 9. 라이트 모드

- 입력창, 셀렉트, 패널, 빈 상태 박스가 어둡게 남으면 안 된다.
- 라이트 모드에서는 대비가 낮은 회색 텍스트를 남용하지 않는다.
- placeholder와 보조 텍스트도 읽히는 수준의 대비를 유지한다.

## 10. 모바일

- 모바일에서도 제목과 버튼이 지나치게 크거나 작아지면 안 된다.
- 번호 공은 가로 흐름이 우선이지만, 좁을 때는 자연스럽게 줄바꿈되게 한다.
- 상단 CTA와 주요 정보는 첫 화면 안에 최대한 들어오게 한다.
- 가로 스크롤은 정말 필요한 영역에만 허용한다.

## 11. 문서 반영 규칙

- 화면 구조가 바뀌면 `Design_Guide`, `Product_Spec`, `history`를 함께 본다.
- 디자인 기준이 바뀌면 가이드 문서에 먼저 적고 코드에 맞춘다.
- 공통 규칙은 문서에 남기지 않으면 완료로 보지 않는다.
