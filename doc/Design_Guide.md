# Design Guide v1.1

## 1. 디자인 목표

Lotto Maker Lab의 디자인은 `한눈에 들어오는 깔끔함`, `빠른 가독성`, `추천/결과/통계의 명확한 역할 분리`를 기준으로 한다.

사용자가 처음 들어왔을 때 아래 세 가지를 즉시 이해해야 한다.

1. 이 사이트가 무엇을 하는지
2. 어디를 눌러야 하는지
3. 최신 회차와 핵심 통계를 어디서 보는지

## 2. 기본 원칙

- 정보는 많아도 화면은 복잡하게 보이지 않게 구성한다.
- 첫 화면의 핵심은 `추천기 진입`과 `최신 결과 확인`이다.
- 카드 수를 늘리기보다 섹션 위계를 명확히 하는 편이 우선이다.
- 광고는 콘텐츠보다 먼저 보이면 안 된다.

## 3. 시선 흐름

### 3-1. 홈

1. 브랜드와 서비스 설명
2. 추천기 CTA
3. 최신 회차 카드
4. 빠른 이동
5. 분석 허브 / 가이드
6. 서비스 설명과 정책 링크

### 3-2. 추천기

1. 전략 선택
2. 조건 입력
3. 결과 생성
4. 결과 이유 문구
5. 통계/상세 이동

### 3-3. 통계 허브

1. 요약 카드
2. 비교 섹션
3. 차트
4. 상세 링크

## 4. 가독성 기준

- 본문 폭은 과도하게 넓히지 않는다.
- 카드 안에는 한 가지 핵심 메시지만 둔다.
- 제목, 설명, CTA는 3단 위계가 명확해야 한다.
- 긴 설명은 2~3문장 단위로 끊는다.
- 차트와 숫자 카드는 같은 화면에서 서로 경쟁하지 않도록 배치한다.

## 5. 레이아웃 기준

- 모바일 우선 단일 컬럼을 기본으로 한다.
- 데스크톱에서는 2단 또는 3단 그리드만 사용한다.
- 상단 히어로와 최신 회차는 항상 첫 화면에서 함께 보여야 한다.
- 섹션 간 간격은 충분히 두어 카드 덩어리가 섞여 보이지 않게 한다.

## 6. 카드 설계 기준

- 카드당 핵심 제목은 1개
- 보조 설명은 짧고 명확하게
- 클릭 가능한 카드는 hover 상태가 분명해야 한다
- 같은 유형의 카드끼리는 높이와 여백을 통일한다

## 7. 타이포그래피

- 제목은 강한 대비를 주고, 설명은 한 단계 낮춘다.
- 한글 본문은 과도하게 압축된 줄간격을 피한다.
- 영어 UI는 보조적 역할로만 쓰고, 기본 노출은 한글 중심으로 유지한다.

## 8. 다크/라이트 모드

- 다크 모드와 라이트 모드 모두 텍스트 가독성을 유지해야 한다.
- 강조색은 두 모드 모두에서 CTA와 핵심 지표를 구분할 수 있어야 한다.
- 색상 변화보다 구조 변화가 더 우선이다.

## 9. 한영 전환

- URL은 영문 유지
- 제목, 설명, 본문은 한글 우선
- 영어 전환 시 의미가 자연스럽게 유지되어야 하며 직역 느낌을 줄인다

## 10. 광고 배치

- 광고는 CTA 바로 앞에 두지 않는다.
- 광고에는 반드시 `광고` 라벨을 둔다.
- 광고와 콘텐츠 카드의 시각적 위계를 분리한다.

## 11. 접근성

- 링크와 버튼은 키보드로 접근 가능해야 한다.
- 번호 볼, 보너스 번호, 차트 항목은 읽을 수 있는 라벨이 있어야 한다.
- 색상만으로 의미를 전달하지 않는다.

## 12. 품질 체크

- 홈이 복잡하게 보이지 않는가
- 첫 화면에 최신 회차가 바로 보이는가
- 추천기 CTA가 가장 먼저 눈에 들어오는가
- 카드와 차트가 서로 경쟁하지 않는가
- 모바일에서도 줄바꿈과 여백이 자연스러운가
