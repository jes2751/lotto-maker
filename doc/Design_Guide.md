# Design Guide v1.10

## 1. 방향 (Premium UI/UX)

- 첫 화면에서 가장 먼저 보여줄 정보는 `최신 당첨번호`, `번호 생성`, `핵심 통계`다.
- 사용자가 바로 행동할 수 있어야 하므로 긴 설명보다 짧은 제목, 짧은 본문, 명확한 버튼이 우선이다.
- 내부 운영 언어는 화면에 노출하지 않는다. (예: 검색 유입, 재방문, 랜딩)
- **배경 및 테마**: 가볍거나 들뜨지 않는 심연의 네이비/블랙(Deep Space) 기반에, 차분하고 신뢰감 있는 스포트라이트 그레이디언트가 아주 느리게 교차하며 전문가적인(FinTech) 무게감을 제공한다.
- **컴포넌트 텍스처**: 정교하고 치밀하게 세공된 금속/유리 질감(Micro-Glassmorphism)을 사용. 1px의 정밀한 화이트 빛반사 라인과 극한의 블러를 통해 단단하고 신뢰할 수 있는 느낌을 극대화한다.

## 1-1. 8/10 품질 기준

- 첫 화면은 `하나의 히어로`로 읽혀야 한다. 최신 회차 블록과 서비스 소개 블록이 각각 따로 메인처럼 보이면 안 된다.
- 한 섹션은 한 가지 역할만 가져야 한다. `설명`, `요약`, `이동`, `통계`를 한 카드 안에 섞지 않는다.
- 같은 카드 스킨이 연속으로 3개 이상 반복되면 완성도가 떨어진다. 정보의 성격이 다르면 레이아웃도 달라져야 한다.
- 사용자가 3초 안에 알아야 할 것은 세 가지뿐이다.
  - 지금 최신 회차가 무엇인지
  - 어디서 바로 번호를 만들 수 있는지
  - 어떤 통계를 먼저 보면 되는지
- 보조 텍스트라도 읽히지 않으면 장식이다. 다크 모드 패널 위에서는 `text-slate-500` 급의 약한 회색을 기본값으로 쓰지 않는다.
- 헤더는 브랜드 소개 영역이지 본문과 경쟁하는 배너가 아니다. 스티키 헤더는 작고 빠르게 읽혀야 한다.
- 색은 역할이 분명해야 한다.
  - `accent/amber`: 주요 CTA
  - `teal`: 데이터 강조, 링크, 상태 하이라이트
  - `silver`: 큰 제목 또는 한정된 포인트
  - 로또 공 색: 분류용만 사용, 브랜드 색 역할을 가져가면 안 된다

## 1-2. 금지 패턴

- 홈 첫 화면을 위아래로 쪼개 두 개의 메인 패널처럼 보이게 만드는 구성
- 의미가 다른 영역을 모두 `둥근 진한 카드` 하나의 문법으로 반복하는 구성
- 다크 모드 카드 안에서 읽기 어려운 회색 보조 문구 남용
- 스티키 헤더 안에 큰 로고, 긴 설명, 방문자 수, 메뉴를 한 번에 밀어 넣는 구성
- 카드가 실제 상호작용 단위가 아닌데도 모든 링크를 카드로 만드는 구성

## 2. 타이포

### 서비스 전체 기본

- 기본 폰트: `Noto Sans KR`, 영문/숫자 특화 폰트로 `Outfit` 등 모던 기하학 폰트 조합 적용
- 본문 기본 두께: `500`
- 제목 기본 두께: `700~800`
- 영문 로고는 자간을 살짝 주되 과하게 벌리지 않는다.

### 글씨 크기 기준

- 메인 제목:
  - desktop: `clamp(1.95rem, 3vw, 3rem)`
  - mobile: `2rem` 전후
  - line-height: `1.14 ~ 1.2`
  - weight: `800`
- 섹션 제목:
  - desktop: `clamp(1.3rem, 2vw, 1.8rem)`
  - mobile: `1.25rem ~ 1.5rem`
  - line-height: `1.2 ~ 1.28`
  - weight: `700`
- 본문 큰 텍스트:
  - `1rem ~ 1.0625rem`
  - line-height: `1.75 ~ 1.85`
- 본문 기본:
  - `0.95rem ~ 1rem`
  - line-height: `1.7 ~ 1.8`
- 보조 텍스트:
  - `0.875rem ~ 0.9rem`
  - line-height: `1.6 ~ 1.7`
- eyebrow:
  - `0.875rem`
  - uppercase
  - tracking은 과하지 않게 `0.1em` 전후

### 타이포 원칙

- 제목은 되도록 한 줄에 보이게 하고, 안 되면 두 줄 안에서 끝낸다.
- 의미 없는 강한 확대는 피한다.
- 밝은 배경에서는 텍스트 대비를 더 진하게 잡는다.
- 작은 민트색 라벨은 너무 작게 쓰지 않는다.
- 보조 텍스트는 배경과 충분히 분리되어야 한다.
  - dark panel: 기본 `slate-300 ~ slate-400`
  - dark footer/legal: 예외적으로 `slate-400 ~ slate-500`
  - light surface: 기본 `slate-600` 이상
- 수치, 빈도, 주석이 중요한 데이터라면 본문보다 한 단계만 약하게 두고 두 단계 이상 죽이지 않는다.

## 3. 레이아웃

- 콘텐츠 최대 폭: `max-w-6xl`
- 주요 화면 상단 여백: `py-12`
- 패널 간 기본 간격: `gap-6` 또는 `gap-8`
- 패널 내부 패딩:
  - desktop: `p-6 ~ p-7`
  - mobile: `p-5 ~ p-6`
- 패널 모서리:
  - `24px` 전후
- 패널 구분:
  - 너무 많은 작은 카드보다 큰 카드 몇 개가 낫다.
- 티타늄 / 글래스모피즘 텍스처 (Trust 룩):
  - 패널 표면은 극도의 투명도(`bg-white/5` 이하)에 맞춘 강력한 블러 효과를 지니며, 상단/좌측에 1px 빛반사 하이라이트를 통해 금융/보안 서비스 수준의 정밀한 질감을 재현한다.
  - 주요 CTA 버튼과 타이포그래피는 '티타늄 실버' 혹은 세련된 '딥 오션 톤'을 택해, 들뜨지 않고 믿음직한 우아함을 제공한다.

### 카드 사용 규칙

- 카드는 `독립적으로 눌러야 하는 단위`이거나 `명확히 묶인 정보 덩어리`일 때만 쓴다.
- 요약 수치 3개를 모두 카드로 만들기 전에 먼저 `한 줄 KPI`, `칩`, `리스트`, `구분선`으로 해결 가능한지 본다.
- 동일한 반경, 동일한 테두리, 동일한 배경을 여러 섹션에 기계적으로 복붙하지 않는다.
- 카드가 많아질수록 섹션 간 위계는 더 강해져야 한다.
  - 메인 히어로: 가장 큰 덩어리
  - 요약 영역: 카드 2~3개 이내
  - 보조 이동 영역: 텍스트 링크 또는 낮은 강조 카드

## 3-1. 핵심 요소 (로또 공)
- **절제된 3D(Resin/Glass) 질감**: 억지스러운 튀어오름 대신, 정밀하게 가공된 당구공이나 보석 느낌의 입체 내부 그림자와 뚜렷한 빛 반사 하이라이트(Highlight)를 얹어 무겁고 단단한 느낌을 준다.
- **정교한 인터랙션**: 방정맞게 튀어 오르는 요소를 배제하고, 마우스 호버 시 빛 반사가 미세하게 강해지고 아주 약간만 커지는 수준의 절제미(Understated precision)를 지향한다.

## 4. 홈 화면

- 첫 화면은 `단일 히어로`가 기본이다.
  - 좌측: 브랜드, 메인 제목, 짧은 설명, CTA 1~2개
  - 우측: 최신 회차, 날짜, 번호, 상세 링크
  - mobile: 위아래 스택으로 접되 하나의 덩어리처럼 읽혀야 한다
- 최신 회차 블록은 `지금 이 서비스가 살아 있다`는 신뢰 증거 역할을 해야 한다.
- 홈 첫 화면에서 설명보다 행동이 먼저 보여야 한다.
  - 1순위 CTA: 번호 생성
  - 2순위 CTA: 회차 조회 또는 통계
- 모바일 히어로에서는 CTA를 한 번에 2개까지만 강하게 노출한다.
  - 3번째 이동 버튼이 필요하면 아래 섹션으로 내리거나 desktop 전용으로 둔다
- 히어로 아래 섹션은 최대 3묶음으로 제한한다.
  - 빠른 시작
  - 핵심 요약
  - 가이드/서비스 안내
- `빠른 시작`은 이동용 섹션이다. 설명이 길어지면 안 된다.
- `핵심 요약`은 데이터 섹션이다. 카드형 수치를 많이 늘리지 않는다.
- `가이드/서비스 안내`는 홈의 주인공이 아니므로 시각적 강도를 한 단계 낮춘다.

## 5. 생성기

- 상단 2단은 좌우 패널 끝선이 맞아야 한다.
- 오른쪽 끝에 애매한 빈 여백이 남으면 안 된다.
- 전략 선택은 한 화면에서 가로 흐름으로 비교되게 한다.
- `filter` 전략일 때만 필터 입력 UI를 보여준다.
- 생성 결과가 없을 때도 화면이 무겁지 않게 유지한다.
- 보너스 번호는 항상 포함한다.

## 6. 통계

- 통계 화면은 `요약 -> 핵심 번호 -> 비교 -> 차트 -> 해석` 순으로 읽히게 한다.
- 숫자 카드 제목은 한 줄에 가깝게 짧게 유지한다.
- 칩 링크는 너무 작지 않게 하고, 클릭 가능하다는 느낌을 줘야 한다.

## 7. 생성 통계

- 생성 통계는 커뮤니티처럼 보이면 안 된다.
- `전략 성과`, `현재 회차 현황`, `적중 분포`, `최근 생성 번호` 순으로 정리한다.
- 적중 분포 카드가 억지로 세로로 늘어나지 않게 한다.
- 전략 점유율, 최고 적중 수, 많이 생성된 번호는 한눈에 보여야 한다.
- 모바일에서는 상단 기준 정보와 KPI를 `2열` 또는 `3열`로 압축해 세로 길이를 먼저 줄인다.
- 모바일에서도 같은 성격의 작은 수치 카드가 4개 이상 세로로 연속되면 다시 묶는 방식을 바꾼다.

## 8. 헤더와 푸터

- 헤더:
  - 로고 > 짧은 설명 > 내비게이션 순서
  - 설명은 desktop 한 줄, mobile 축약 또는 숨김까지 허용
  - 메뉴는 본문보다 약간 크게, 그러나 헤더 높이를 늘릴 정도로 크면 안 된다
  - 방문자 수 같은 보조 정보는 헤더의 주 역할을 방해하지 않는 낮은 우선순위로 둔다
  - 스티키 상태에서도 첫 화면 콘텐츠가 과도하게 가려지지 않아야 한다
- 푸터:
  - 소개 영역 + 링크 영역
  - 링크는 `서비스`, `정책`으로 나누되 박스는 하나로 유지
  - 푸터는 정보 정리용이지 시선을 끄는 영역이 아니다
  - 소개/법적 문구 대비가 너무 낮아 읽기 힘들어지면 안 된다

## 9. 라이트 모드

- 입력창, 셀렉트, 패널, 빈 상태 박스가 어둡게 남으면 안 된다.
- 라이트 모드에서는 대비가 낮은 회색 텍스트를 남용하지 않는다.
- placeholder와 보조 텍스트도 읽히는 수준의 대비를 유지한다.
- 다크 모드에서 보이던 실버/민트 강조가 라이트 모드에서 과하게 떠 보이면 채도를 낮춘다.
- 같은 컴포넌트가 테마에 따라 전혀 다른 위계로 보이면 안 된다.

## 10. 모바일

- 모바일에서도 제목과 버튼이 지나치게 크거나 작아지면 안 된다.
- 번호 공은 가로 흐름이 우선이지만, 좁을 때는 자연스럽게 줄바꿈되게 한다.
- 상단 CTA와 주요 정보는 첫 화면 안에 최대한 들어오게 한다.
- 가로 스크롤은 정말 필요한 영역에만 허용한다.
- 모바일 첫 화면에서 헤더가 히어로보다 더 많은 높이를 차지하면 실패다.
- 홈의 보조 링크 섹션은 모바일에서 카드보다는 리스트에 가깝게 가볍게 접히는 편이 낫다.
- 통계와 수치 카드는 모바일에서 숫자와 라벨이 한 번에 읽히는 높이로 유지한다.

## 10-1. 우선 개선 항목

- 홈 첫 화면을 `단일 히어로`로 재구성한다.
- `빠른 시작`, `핵심 요약`, `가이드/서비스 안내`의 카드 밀도를 줄인다.
- 다크 모드에서 보조 텍스트와 빈도 수치 대비를 한 단계 올린다.
- 스티키 헤더의 높이와 설명 길이를 줄인다.
- 색 역할을 고정하고, 브랜드 강조색과 번호 공 분류색이 충돌하지 않게 정리한다.

## 11. 문서 반영 규칙

- 화면 구조가 바뀌면 `Design_Guide`, `Product_Spec`, `history`를 함께 본다.
- 디자인 기준이 바뀌면 가이드 문서에 먼저 적고 코드에 맞춘다.
- 공통 규칙은 문서에 남기지 않으면 완료로 보지 않는다.

## 12. 지금까지 작업한 디자인 검토 방법

### 12-1. 검토 목적

- 이번 리디자인의 목표는 `신뢰형 로또 컨트롤룸` 톤이 실제 화면에서 일관되게 읽히는지 확인하는 것이다.
- 검토는 예쁜지 아닌지보다 아래 4가지를 먼저 본다.
  - 첫 화면에서 무엇을 해야 하는지 바로 보이는가
  - 같은 카드가 기계적으로 반복되지 않는가
  - 보조 정보와 수치가 실제로 읽히는가
  - 홈, 통계, 결과, 분석, 생성 통계가 같은 제품처럼 느껴지는가

### 12-2. 실행 순서

- 로컬 실행:
  - `npm run dev`
- 우선 검토 경로:
  - `/`
  - `/generate`
  - `/stats`
  - `/generated-stats`
  - `/draws`
  - `/draws/[round]`
  - `/latest-lotto-results`
  - `/draw-analysis`
  - `/draw-analysis/[round]`
- 가능하면 아래 3개 뷰포트에서 본다.
  - mobile: `390px` 전후
  - tablet: `768px` 전후
  - desktop: `1280px` 이상

### 12-3. 공통 체크리스트

- 첫 화면:
  - 제목, 최신 회차, 1순위 CTA가 3초 안에 구분되어야 한다.
  - 히어로가 카드 모음처럼 보이면 실패다.
- 카드 밀도:
  - 같은 `둥근 진한 카드`가 연속해서 여러 줄 반복되면 다시 줄인다.
  - 이동용 링크는 가능한 한 `리스트`나 `낮은 강조 카드`로 읽혀야 한다.
- 대비:
  - 숫자, 빈도, 날짜, 설명이 어두운 배경에서 흐려 보이지 않아야 한다.
  - 보조 텍스트를 읽으려고 눈을 더 써야 하면 대비가 부족한 것이다.
- 일관성:
  - CTA는 `accent/amber`, 데이터 강조는 `teal`, 큰 제목 포인트는 `silver` 역할을 유지해야 한다.
  - 상태 배지, 칩 링크, KPI 셀의 톤이 페이지마다 크게 흔들리면 안 된다.
- 모바일:
  - 헤더가 첫 화면을 과도하게 잡아먹지 않아야 한다.
  - 링크 영역이 모바일에서 무거운 카드 더미처럼 보이면 다시 가볍게 만든다.

### 12-4. 페이지별 확인 포인트

- 홈 `/`
  - 최신 회차와 서비스 소개가 하나의 히어로로 읽히는지 본다.
  - `빠른 시작`, `핵심 요약`, `가이드/서비스 안내`가 서로 다른 역할로 구분되는지 본다.
- 생성기 `/generate`
  - 상단 2단이 한 줄처럼 안정적으로 보이는지, 오른쪽 끝이 비지 않는지 본다.
- 통계 `/stats`
  - `요약 -> 핵심 번호 -> 비교 -> 차트/해석` 흐름이 한 번에 읽히는지 본다.
- 생성 통계 `/generated-stats`
  - 커뮤니티 피드가 아니라 `공개 실험실`처럼 읽히는지 본다.
  - 전략 성과, 점유율, 적중 분포가 각각 다른 역할로 분리되어야 한다.
- 회차 조회 `/draws`, `/draws/[round]`
  - 검색/탐색 화면과 상세 화면의 역할 차이가 분명해야 한다.
  - 상세 화면에서 최신 배지, 이동 링크, 요약 정보가 한 덩어리로 뭉개지면 안 된다.
- 최신 결과 `/latest-lotto-results`
  - 최신 회차 확인 후 상세/분석으로 이동하는 동선이 짧아야 한다.
- 분석 허브 `/draw-analysis`, `/draw-analysis/[round]`
  - 허브는 탐색용, 상세는 해석용으로 읽혀야 한다.
  - 관련 링크가 또 다른 카드 벽처럼 보이면 줄여야 한다.
  - 허브의 빠른 탐색 카드 수는 기본 6개 안쪽으로 두고, 더 깊은 이동은 보조 링크로 넘긴다.
  - 상세 하단의 관련 링크는 `카드 + 칩 + 리스트`를 섞어 밀도를 분산한다.
- 공유 카드/OG
  - 제품 본체와 다른 브랜드처럼 보이면 실패다.
  - 보라 템플릿 느낌이 남아 있지 않은지 확인한다.

### 12-5. 검토 통과 기준

- 아래 6개 중 5개 이상이 `예`여야 1차 통과로 본다.
  - 첫 화면의 주인공이 하나로 읽힌다.
  - CTA 우선순위가 분명하다.
  - 보조 텍스트와 수치가 쉽게 읽힌다.
  - 카드 반복이 줄고 섹션 역할 차이가 보인다.
  - 모바일에서 링크/가이드 영역이 과하게 무겁지 않다.
  - 생성 통계, 결과, 분석, 공유 카드가 같은 브랜드 언어를 쓴다.

### 12-6. 검토 후 문서 업데이트 규칙

- 검토 중 새 규칙이 생기면 먼저 이 문서에 추가한다.
- 실제 수정이 끝나면 `history.md`에 화면 단위로 남긴다.
- 검토 결과가 플랜 변경까지 이어지면 `plan.md`에도 반영한다.
