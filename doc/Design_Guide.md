# Design Guide v1.2

## 1. 목표

Lotto Maker Lab의 화면은 `빠르게 이해되는 구조`, `낮은 피로도`, `즉시 행동 가능한 UI`를 우선한다.  
첫 화면에서 사용자가 해야 할 일은 세 가지다.

1. 최신 당첨번호를 본다.
2. 번호를 생성한다.
3. 핵심 통계를 본다.

그 외 정보는 모두 아래로 내리거나 별도 페이지로 보낸다.

## 2. 핵심 원칙

### 2-1. 한눈에 들어오는 구조
- 한 화면 안에서 경쟁하는 핵심 CTA는 최대 2개까지만 둔다.
- 첫 화면의 핵심 섹션은 3개를 넘기지 않는다.
- 카드가 많아지면 정보가 많아진 것이 아니라 판단 비용이 높아진 것으로 본다.

### 2-2. 가독성 우선
- 긴 설명보다 짧은 제목과 한 줄 요약을 우선한다.
- 한 카드 안에는 메시지 하나만 둔다.
- 본문 텍스트는 짧은 문장, 충분한 줄간격, 좁은 폭을 유지한다.

### 2-3. 정보 위계
- 최신성, 행동, 요약 순으로 위계를 잡는다.
- 홈은 `최신 결과 -> 생성 -> 핵심 요약` 순서를 유지한다.
- 정책, FAQ, 운영 정보는 푸터나 하단 블록에서만 다룬다.

## 3. 페이지별 우선순위

### 3-1. 홈
1. 최신 당첨번호
2. 번호 생성 CTA
3. 핵심 통계 요약
4. 생성 통계 / 회차 분석 / FAQ 링크

홈에서 제거하거나 축소할 것:
- 과도한 가이드 카드
- 긴 서비스 소개
- 중복 CTA
- 정책 링크의 과도한 전면 배치

### 3-2. 번호 생성기
1. 생성기 패널
2. 현재 회차 기준 안내
3. 생성 전략 설명

생성기 페이지는 “읽는 페이지”보다 “쓰는 페이지”에 가깝게 유지한다.

### 3-3. 통계 대시보드
1. 비교 기준 선택
2. 핵심 숫자 요약
3. 차트
4. 해석 가이드

통계 페이지는 많은 섹션보다 `요약 -> 비교 -> 세부` 흐름을 유지한다.

### 3-4. 생성 통계 허브
1. 이번 회차 생성 수
2. 전략 성과 보드
3. 많이 생성된 번호
4. 최근 생성 번호

게시판처럼 보이지 않도록 카드 수를 제한하고, 공개 현황 페이지처럼 구성한다.

## 4. 레이아웃 기준

- 모바일 우선 단일 컬럼을 기본으로 설계한다.
- 데스크톱은 2컬럼까지 허용하고, 3컬럼 이상은 요약 카드 영역에만 제한적으로 사용한다.
- 섹션 간 간격은 넓게 두고, 카드 내부 간격은 일관되게 유지한다.
- 카드가 많아질수록 배경 대비보다 간격과 정렬로 구분한다.

## 5. 타이포 기준

- 제목은 짧고 강하게, 설명은 짧고 부드럽게 쓴다.
- 한글 기준 제목은 1~2줄 안에서 끝내는 것을 기본으로 한다.
- 영문은 보조 정보에만 사용하고, 한글 사용자에게 필요한 정보는 항상 한글이 먼저 보여야 한다.

## 6. CTA 기준

- 페이지별 대표 CTA는 1개, 보조 CTA는 1개까지만 둔다.
- CTA 텍스트는 행동 중심으로 작성한다.
  - 예: `번호 생성하기`, `회차 상세 보기`, `통계 보기`
- 설명형 링크는 CTA처럼 보이지 않게 구분한다.

## 7. 차트와 데이터 카드

- 차트는 “한 번에 하나의 질문”에만 답해야 한다.
- 차트 옆에 긴 설명 문단을 붙이지 않는다.
- 숫자 카드에는 지표명, 값, 짧은 보조 문장만 둔다.

## 8. 광고와 정책 정보

- 광고는 핵심 CTA 앞에 두지 않는다.
- 광고와 콘텐츠는 시각적으로 명확히 분리한다.
- 개인정보처리방침, 이용약관, 문의는 신뢰 정보이지만 핵심 행동을 방해하지 않게 하단으로 보낸다.

## 9. 다크/라이트 모드

- 다크 모드에서도 카드 구분은 색보다 간격과 테두리로 해결한다.
- 라이트 모드는 밝아도 정보 밀도가 높아 보이지 않도록 섹션 수를 줄이는 것이 우선이다.

## 10. 품질 체크

- 첫 화면에서 3초 안에 `무엇을 할 수 있는지`가 보여야 한다.
- 같은 내용을 다른 카드에서 반복하지 않는다.
- 한 섹션을 지워도 페이지 목적이 유지되면, 그 섹션은 기본적으로 제거 후보로 본다.
