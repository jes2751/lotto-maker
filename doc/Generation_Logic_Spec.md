# 생성 로직 명세 v1

## 1. 목적

이 문서는 `random`, `frequency`, `mixed` 전략의 입력, 처리, 출력 규칙을 고정한다. 구현자는 이 문서를 기준으로 동일한 결과 형태와 검증 기준을 유지해야 한다.

## 2. 공통 규칙

* 메인 번호는 항상 1~45 범위에서 6개를 생성한다.
* 메인 번호는 중복되면 안 된다.
* 반환 전 메인 번호는 오름차순으로 정렬한다.
* `count`는 1~5만 허용한다.
* `include_bonus`가 `true`이면 보너스 번호 1개를 추가한다.
* 보너스 번호는 1~45 범위여야 하며 메인 6개와 중복되면 안 된다.
* 출력 필드는 `id`, `strategy`, `numbers`, `bonus?`, `reason`, `generatedAt`로 고정한다.

## 3. 입력 계약

요청 필드:

* `strategy`: `random | frequency | mixed`
* `count`: 정수 1~5
* `include_bonus`: boolean

유효하지 않은 입력은 생성 로직을 실행하지 않고 `VALIDATION_ERROR`를 반환한다.

## 4. 전략별 규칙

### 4.1 `random`

목적:

* 완전 무작위 조합을 제공한다.

알고리즘:

1. 1~45 전체 풀을 만든다.
2. 중복 없이 6개를 무작위 추출한다.
3. 오름차순으로 정렬한다.
4. `include_bonus`가 `true`이면 남은 풀에서 1개를 무작위 추출한다.

이유 문구:

* `완전 랜덤 기준으로 중복 없이 조합했습니다.`

### 4.2 `frequency`

목적:

* 시드 데이터에서 자주 등장한 메인 번호를 더 자주 뽑히게 한다.

가중치 규칙:

* 기본 가중치는 모든 번호에 `1`을 부여한다.
* 시드 데이터의 각 회차 메인 번호가 한 번 등장할 때마다 해당 번호 가중치에 `+1`을 더한다.
* 보너스 번호는 가중치 계산에 포함하지 않는다.

알고리즘:

1. 시드 전체 회차의 메인 번호 빈도로 가중치 맵을 만든다.
2. 1~45 풀에서 가중치 기반 랜덤 선택을 수행한다.
3. 이미 선택된 번호는 다음 선택 대상에서 제외한다.
4. 6개를 고른 뒤 오름차순 정렬한다.
5. `include_bonus`가 `true`이면 남은 풀에서 1개를 무작위 추출한다.

이유 문구:

* `시드 회차에서 자주 나온 번호에 가중치를 둬 조합했습니다.`

### 4.3 `mixed`

목적:

* 빈도 기반 편향과 완전 무작위 성격을 함께 제공한다.

알고리즘:

1. `frequency` 방식으로 3개를 먼저 선택한다.
2. 이미 선택한 번호를 제외한 나머지 풀에서 `random` 방식으로 3개를 선택한다.
3. 두 결과를 합친 뒤 오름차순 정렬한다.
4. `include_bonus`가 `true`이면 남은 풀에서 1개를 무작위 추출한다.

이유 문구:

* `빈도 가중치와 랜덤 조합을 섞어 균형 있게 구성했습니다.`

## 5. 예외 처리

* `strategy`가 허용 값이 아니면 실패한다.
* `count`가 정수가 아니거나 범위를 벗어나면 실패한다.
* 시드 데이터가 비어 있어도 `random`과 `mixed`는 동작 가능해야 한다.
* 시드 데이터가 비어 있으면 `frequency`는 기본 가중치 `1`만 사용한 무작위 선택처럼 동작한다.

## 6. 테스트 기준

필수 검증:

* 각 세트는 정확히 6개의 메인 번호를 가져야 한다.
* 메인 번호는 모두 1~45 범위여야 한다.
* 메인 번호는 중복되면 안 된다.
* 반환 전 메인 번호는 오름차순이어야 한다.
* 보너스 번호는 메인 번호와 중복되면 안 된다.
* 전략별 `reason` 문구는 고정 템플릿과 일치해야 한다.

예시 시나리오:

* `random`, `count=2`, `include_bonus=true`면 2세트와 2개의 보너스가 반환된다.
* `frequency`, `count=1`, `include_bonus=false`면 보너스 필드가 없어야 한다.
* `mixed`, `count=0`이면 `VALIDATION_ERROR`가 반환되어야 한다.
